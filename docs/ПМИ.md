## 1. Вход в систему

### Цель
Авторизоваться в системе для получения доступа к функциям управления кормлением.

### Предусловия
- Система развернута и запущена
- У пользователя есть учетные данные (логин и пароль)
- Браузер открыт и доступен

### Шаги выполнения

**Шаг 1.1:** Откройте браузер.

**Шаг 1.2:** В адресной строке браузера введите адрес системы, который вы получили от разработчиков.

**Шаг 1.3:** Нажмите клавишу `Enter` на клавиатуре.

**Ожидаемый результат:** Откроется страница входа в систему с полями для ввода логина и пароля.

**Шаг 1.4:** На странице входа в центре экрана находится поле с надписью "Имя пользователя".

**Шаг 1.5:** Кликните левой кнопкой мыши в поле "Имя пользователя".

**Шаг 1.6:** Введите свое имя пользователя с клавиатуры.

**Шаг 1.7:** Под полем "Имя пользователя" находится поле с надписью "Пароль".

**Шаг 1.8:** Кликните левой кнопкой мыши в поле "Пароль".

**Шаг 1.9:** Введите свой пароль с клавиатуры (символы скрыты точками).

**Шаг 1.10:** В нижней части формы входа находится кнопка "Войти" (синяя кнопка справа).

**Шаг 1.11:** Кликните левой кнопкой мыши на кнопку "Войти".

**Ожидаемый результат:** 
- Если имя пользователя и пароль верны, произойдет перенаправление на главную страницу системы
- Если имя пользователя или пароль неверны, появится сообщение об ошибке "Неверные учетные данные"

**Шаг 1.12:** Проверьте, что вы находитесь на главной странице системы. Это видно по наличию меню навигации и заголовка "Главная".

### Пост-условия
- Пользователь авторизован в системе
- Открыта главная страница системы
- Доступны все функции управления кормлением

## 2. Создание задачи кормления

### Цель
Создать новую задачу кормления для указанного бассейна с заданными параметрами.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница "Управление кормлением"
- В системе есть хотя бы один бассейн
- В системе есть хотя бы один тип корма
- В системе есть хотя бы один период кормления

### Шаги выполнения

**Шаг 2.1:** В меню навигации слева кликните левой кнопкой мыши на пункт "Кормление".

**Ожидаемый результат:** Откроется страница со списком задач кормления.

**Шаг 2.2:** В верхней части страницы "Кормление" находится кнопка "Создать задачу" (синяя кнопка).

**Шаг 2.3:** Кликните левой кнопкой мыши на кнопку "Создать задачу".

**Ожидаемый результат:** Откроется модальное окно "Создание задачи кормления" с полями для ввода.

**Шаг 2.4:** В модальном окне в верхней части формы находится поле выбора бассейна с надписью "Бассейн".

**Шаг 2.5:** Кликните левой кнопкой мыши на поле выбора бассейна.

**Ожидаемый результат:** Откроется список доступных бассейнов.

**Шаг 2.6:** Выберите нужный бассейн из списка, кликнув на него левой кнопкой мыши.

**Ожидаемый результат:** В поле будет отображаться выбранный бассейн.

**Шаг 2.7:** Под полем "Бассейн" находится поле выбора корма с надписью "Корм".

**Шаг 2.8:** Кликните левой кнопкой мыши на поле выбора корма.

**Ожидаемый результат:** Откроется список доступных типов корма.

**Шаг 2.9:** Выберите нужный тип корма из списка, кликнув на него левой кнопкой мыши.

**Ожидаемый результат:** В поле будет отображаться выбранный тип корма.

**Шаг 2.10:** Под полем "Корм" находится поле ввода массы корма с надписью "Масса (кг)".

**Шаг 2.11:** Кликните левой кнопкой мыши в поле ввода массы.

**Шаг 2.12:** Введите числовое значение массы корма в килограммах с клавиатуры (например, `0.1` для 100 грамм или `1.5` для 1.5 килограмм).

**Ожидаемый результат:** В поле отображается введенное значение.

**Шаг 2.13:** Под полем "Масса (кг)" находится поле выбора периода кормления с надписью "Период".

**Шаг 2.14:** Кликните левой кнопкой мыши на поле выбора периода.

**Ожидаемый результат:** Откроется список доступных периодов кормления с опцией "Другое".

**Шаг 2.15:** Выберите период из списка, кликнув на него левой кнопкой мыши. Если вы выбрали "Другое", появится дополнительное поле "Период (другое)" для ввода времени вручную.

**Ожидаемый результат:** В поле будет отображаться выбранный период. Если выбрано "Другое", отобразится поле для ввода времени.

**Шаг 2.16:** Если вы выбрали "Другое" в поле "Период", введите время кормления в поле "Период (другое)" в формате ЧЧ:ММ (например, `14:30`).

**Ожидаемый результат:** В поле отображается введенное время.

**Шаг 2.17:** В нижней части модального окна находятся две кнопки: "Отменить" (слева) и "Создать" (справа, синяя кнопка).

**Шаг 2.18:** Кликните левой кнопкой мыши на кнопку "Создать".

**Ожидаемый результат:** 
- Если все поля заполнены корректно, задача будет создана
- Модальное окно закроется
- Новая задача появится в списке задач кормления
- Если есть ошибки валидации, появятся сообщения об ошибках под соответствующими полями

**Шаг 2.19:** Проверьте, что новая задача отображается в списке задач кормления.

### Пост-условия
- Создана новая задача кормления
- Задача отображается в списке с статусом "Запланировано"
- Все введенные данные сохранены корректно

## 3. Просмотр списка задач кормления

### Цель
Просмотреть список всех созданных задач кормления.

### Предусловия
- Пользователь авторизован в системе
- В системе может быть хотя бы одна задача кормления (список может быть пустым)

### Шаги выполнения

**Шаг 3.1:** В меню навигации слева кликните на пункт "Кормление".

**Шаг 3.2:** Кликните левой кнопкой мыши на пункт "Кормление" в меню.

**Ожидаемый результат:** Откроется страница со списком задач кормления.

**Шаг 3.3:** Просмотрите таблицу со списком задач. Таблица должна содержать следующие колонки:
- Бассейн
- Корм
- Масса (кг)
- Период
- Кнопка удаления

**Шаг 3.4:** Если задач много и они не помещаются на одной странице, в нижней части таблицы находятся элементы пагинации (кнопки "Следующая", "Предыдущая", номера страниц).

**Шаг 3.5:** Для перехода на следующую страницу кликните на кнопку "Следующая" или номер следующей страницы в пагинации внизу таблицы.

**Ожидаемый результат:** Отобразятся задачи со следующей страницы.

**Шаг 3.6:** Для перехода на предыдущую страницу кликните на кнопку "Предыдущая" или номер предыдущей страницы в пагинации внизу таблицы.

**Ожидаемый результат:** Отобразятся задачи с предыдущей страницы.

**Шаг 3.7:** Для изменения количества задач на странице в пагинации внизу таблицы на странице "Кормление" кликните на выпадающий список с количеством элементов (например, "10", "20", "50").

**Шаг 3.8:** Кликните на выпадающий список в пагинации.

**Шаг 3.9:** Выберите нужное количество задач на странице.

**Ожидаемый результат:** Количество отображаемых задач изменится согласно выбранному значению.

### Пост-условия
- Отображен список всех задач кормления
- Пользователь может видеть все доступные задачи

## 4. Редактирование задачи кормления

### Цель
Изменить параметры существующей задачи кормления.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть хотя бы одна задача кормления

### Шаги выполнения

**Шаг 4.1:** Откройте раздел "Кормление" (в меню навигации слева кликните на пункт "Кормление") и в таблице найдите строку с задачей кормления, которую нужно отредактировать.

**Шаг 4.2:** В правой части строки задачи находится кнопка редактирования (иконка карандаша), или дважды кликните на ячейку, которую нужно изменить.

**Шаг 4.3:** Кликните левой кнопкой мыши на кнопку редактирования или дважды кликните на ячейку.

**Ожидаемый результат:** Ячейка станет редактируемой.

**Шаг 4.4:** Для редактирования ячейки:
- Редактируемыми являются следующие поля: "Бассейн", "Корм", "Масса (кг)", "Период"
- Кликните на ячейку, которую нужно изменить
- Введите новое значение или выберите из списка (для полей "Бассейн", "Корм", "Период" с выпадающим списком)
- Для поля "Масса (кг)" введите числовое значение с клавиатуры
- Нажмите клавишу `Enter` или кликните вне ячейки для сохранения

**Шаг 4.5:** Изменения сохраняются автоматически после нажатия `Enter` или клика вне ячейки.

**Ожидаемый результат:** 
- Изменения сохранены
- Появится сообщение об успешном сохранении (если предусмотрено)
- Обновленные данные отображаются в таблице
- Если есть ошибки валидации, появятся сообщения об ошибках

**Шаг 4.6:** Проверьте, что изменения отображаются в списке задач.

### Пост-условия
- Задача кормления обновлена с новыми параметрами
- Изменения сохранены в системе
- Обновленные данные отображаются в списке

## 5. Удаление задачи кормления

### Цель
Удалить задачу кормления из системы.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть хотя бы одна задача кормления

### Шаги выполнения

**Шаг 5.1:** Откройте раздел "Кормление" (в меню навигации слева кликните на пункт "Кормление") и в таблице найдите строку с задачей кормления, которую нужно удалить.

**Шаг 5.2:** В правой части строки задачи находится кнопка удаления (иконка корзины).

**Шаг 5.3:** Кликните левой кнопкой мыши на кнопку удаления.

**Ожидаемый результат:** Откроется диалоговое окно с подтверждением удаления задачи.

**Шаг 5.4:** В диалоговом окне подтверждения удаления:
- Прочитайте сообщение о подтверждении удаления задачи
- Для подтверждения удаления кликните на кнопку "Удалить" (или "Подтвердить")
- Для отмены удаления кликните на кнопку "Отменить" (или закройте диалоговое окно)

**Ожидаемый результат при подтверждении:** 
- Задача удалена из системы
- Диалоговое окно закроется
- Появится сообщение об успешном удалении (если предусмотрено)
- Задача исчезнет из списка задач
- Если была ошибка, появится сообщение об ошибке

**Ожидаемый результат при отмене:** 
- Диалоговое окно закроется
- Задача останется в списке без изменений

**Шаг 5.5:** Проверьте, что удаленная задача больше не отображается в списке задач (если было подтверждено удаление).

### Пост-условия
- Задача кормления удалена из системы
- Задача больше не отображается в списке
- Данные удалены из базы данных

## 6. Фильтрация задач кормления

### Цель
Отфильтровать список задач кормления по заданным критериям.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть задачи кормления

### Шаги выполнения

**Шаг 6.1:** На странице "Кормление" над таблицей задач находится панель фильтров с полями для фильтрации.

**Шаг 6.2:** Для фильтрации по бассейну:
- В панели фильтров над таблицей на странице "Кормление" кликните на поле фильтра "Бассейн" (первое поле слева)
- Кликните левой кнопкой мыши на это поле
- Выберите нужный бассейн из выпадающего списка

**Ожидаемый результат:** Список задач обновится и покажет только задачи для выбранного бассейна.

**Шаг 6.3:** Для фильтрации по типу корма:
- В панели фильтров над таблицей на странице "Кормление" кликните на поле фильтра "Корм" (второе поле)
- Кликните левой кнопкой мыши на это поле
- Выберите нужный тип корма из выпадающего списка

**Ожидаемый результат:** Список задач обновится и покажет только задачи с выбранным типом корма.

**Шаг 6.4:** Для фильтрации по массе корма:
- В панели фильтров над таблицей на странице "Кормление" кликните на слайдер фильтра "Масса" (третье поле)
- Переместите левый ползунок слайдера для установки минимального значения массы
- Переместите правый ползунок слайдера для установки максимального значения массы
- Или кликните на нужное значение на шкале слайдера

**Ожидаемый результат:** Список задач обновится и покажет только задачи с массой в указанном диапазоне.

**Шаг 6.5:** Для сброса фильтров:
- Очистите выбранные значения в полях фильтров, кликнув на крестик в каждом поле
- Или обновите страницу

**Ожидаемый результат:** Все фильтры сброшены, отображаются все задачи без фильтрации.

**Шаг 6.6:** Проверьте, что список задач соответствует примененным фильтрам.

### Пост-условия
- Применены выбранные фильтры
- Список задач отображает только задачи, соответствующие критериям фильтрации

## 7. Экспорт данных о кормлении

### Цель
Выгрузить данные о задачах кормления в файл для дальнейшего анализа.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть задачи кормления

### Шаги выполнения

**Шаг 7.1:** На странице "Кормление" в панели инструментов над таблицей находится кнопка экспорта.

**Шаг 7.2:** Кликните левой кнопкой мыши на кнопку экспорта.

**Ожидаемый результат:** 
- Если на странице применены фильтры (выбран бассейн, тип корма или диапазон массы), экспортируются только отфильтрованные данные, соответствующие текущим фильтрам
- Если фильтры не применены (все поля фильтров пусты), экспортируются все данные о задачах кормления из системы
- Начнется загрузка файла в формате CSV

**Шаг 7.3:** Дождитесь завершения загрузки файла. Файл автоматически сохраняется в папку "Загрузки" (Downloads).

**Шаг 7.4:** Откройте папку "Загрузки" (Downloads) на вашем компьютере и найдите загруженный файл `feeding_data.csv`.

**Шаг 7.5:** Откройте файл в программе для работы с таблицами (Microsoft Excel, LibreOffice Calc или текстовом редакторе). Примечание: файл экспортируется в формате CSV и может быть открыт любым текстовым редактором или программой для работы с таблицами.

**Ожидаемый результат:** Файл содержит данные о задачах кормления в формате CSV с колонками:
- Бассейн
- Популяция
- Время
- Масса
- Тип корма

**Шаг 7.6:** Проверьте, что все данные экспортированы корректно и соответствуют данным в системе.

### Пост-условия
- Файл с данными о кормлении сохранен на компьютере пользователя
- Данные в файле соответствуют данным в системе
- Файл можно открыть и проанализировать в сторонних программах

## 8. Просмотр логов системы

### Цель
Просмотреть журнал событий системы для мониторинга работы оборудования и выявления проблем.

### Предусловия
- Пользователь авторизован в системе
- В системе есть записи в журнале логов

### Шаги выполнения

**Шаг 8.1:** В меню навигации слева кликните на пункт "Логи".

**Шаг 8.2:** Кликните левой кнопкой мыши на пункт "Логи" в меню.

**Ожидаемый результат:** Откроется страница со списком логов системы.

**Шаг 8.3:** Просмотрите таблицу со списком логов. Таблица содержит следующие колонки:
- Тип - может быть "Тележка", "Бункер", "Система"
- Действие - описание выполненного действия
- Время - дата и время события
- Описание - дополнительная информация о событии

**Шаг 8.4:** Логи отсортированы по времени, самые свежие сверху.

**Шаг 8.5:** Для просмотра подробной информации о конкретном логе:
- В таблице на странице "Логи" найдите нужную запись в списке (прокрутите таблицу при необходимости)
- Кликните левой кнопкой мыши на строку с логом

**Ожидаемый результат:** Откроется модальное окно с подробной информацией о событии.

**Шаг 8.6:** Для перехода на следующую страницу логов (если их много):
- В нижней части таблицы на странице "Логи" кликните на элементы пагинации
- Кликните на кнопку "Следующая" или номер следующей страницы

**Ожидаемый результат:** Отобразятся логи со следующей страницы.

**Шаг 8.7:** Для изменения количества логов на странице:
- В пагинации внизу таблицы на странице "Логи" кликните на выпадающий список с количеством элементов
- Кликните на него
- Выберите нужное количество

**Ожидаемый результат:** Количество отображаемых логов изменится согласно выбранному значению.

### Пост-условия

- Пользователь может видеть все доступные записи о событиях

## 9. Фильтрация логов

### Цель
Отфильтровать список логов по заданным критериям для поиска нужных событий.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком логов
- В системе есть записи в журнале логов

### Шаги выполнения

**Шаг 9.1:** На странице "Логи" над таблицей находится панель фильтров.

**Шаг 9.2:** Для фильтрации по типу лога:
- В панели фильтров над таблицей на странице "Логи" кликните на поле фильтра "Тип" (первое поле слева)
- Кликните левой кнопкой мыши на это поле
- Выберите нужный тип из выпадающего списка:
  - "Тележка" - логи, связанные с тележкой
  - "Бункер" - логи, связанные с бункером
  - "Система" - логи, связанные с системой

**Ожидаемый результат:** Список логов обновится и покажет только логи выбранного типа.

**Шаг 9.3:** Для фильтрации по дате:
- В панели фильтров над таблицей на странице "Логи" кликните на поля фильтра "Дата от" и "Дата до"
- Кликните левой кнопкой мыши в поле "Дата от"
- Выберите начальную дату из календаря или введите в формате ГГГГ-ММ-ДД (например, `2024-01-01`)
- Кликните левой кнопкой мыши в поле "Дата до"
- Выберите конечную дату из календаря или введите в формате ГГГГ-ММ-ДД

**Ожидаемый результат:** Список логов обновится и покажет только логи за указанный период.

**Шаг 9.4:** Для сброса фильтров:
- Очистите выбранные значения в полях фильтров, кликнув на крестик в каждом поле
- Или обновите страницу

**Ожидаемый результат:** Все фильтры сброшены, отображаются все логи без фильтрации.

**Шаг 9.5:** Проверьте, что список логов соответствует примененным фильтрам.

### Пост-условия
- Применены выбранные фильтры
- Список логов отображает только записи, соответствующие критериям фильтрации

## 10. Комплексная проверка системы кормления

### Цель
Выполнить полную проверку всех компонентов системы кормления в рабочем режиме для подтверждения готовности к эксплуатации.

### Предусловия
- Все предыдущие проверки выполнены успешно
- Система включена и готова к работе
- Пользователь авторизован в веб-интерфейсе

### Шаги выполнения

**Шаг 10.1:** Откройте раздел "Кормление" (в меню навигации слева кликните на пункт "Кормление"), затем кликните на кнопку "Создать задачу" в верхней части страницы. В модальном окне создайте тестовую задачу кормления:
- В поле "Бассейн" выберите бассейн
- В поле "Корм" выберите тип корма
- В поле "Масса (кг)" введите массу корма 0.5
- В поле "Период" выберите период кормления
- Кликните на кнопку "Создать" в нижней части модального окна

**Ожидаемый результат:** Задача создана успешно.

**Шаг 10.2:** Запустите выполнение задачи кормления через веб-интерфейс.

**Ожидаемый результат:** Система начинает выполнение задачи.

**Шаг 10.3:** Откройте раздел "Логи" (в меню навигации слева кликните на пункт "Логи") и проверьте, что все операции зафиксированы:
- Загрузка корма
- Выгрузка корма
- Завершение задачи

**Ожидаемый результат:** Все операции зафиксированы в логах с указанием времени и параметров.

**Шаг 10.4:** Откройте раздел "Кормление" (в меню навигации слева кликните на пункт "Кормление") и в таблице задач проверьте статус выполненной задачи. Убедитесь, что задача отмечена как "Выполнено".

**Ожидаемый результат:** Задача отмечена как выполненная.

### Пост-условия
- Все компоненты системы проверены в рабочем режиме
- Подтверждена готовность системы к эксплуатации
- Все операции выполнены в соответствии с требованиями
- Все данные зафиксированы в системе
