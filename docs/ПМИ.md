## 1. Вход в систему

### Цель
Авторизоваться в системе для получения доступа к функциям управления кормлением.

### Предусловия
- Система развернута и запущена
- У пользователя есть учетные данные (логин и пароль)
- Браузер открыт и доступен

### Шаги выполнения

**Шаг 1.1:** Откройте браузер.

**Шаг 1.2:** В адресной строке браузера введите адрес системы, который вы получили от разработчиков.

**Шаг 1.3:** Нажмите клавишу `Enter` на клавиатуре.

**Ожидаемый результат:** Откроется страница входа в систему с полями для ввода логина и пароля.

**Шаг 1.4:** На странице входа в центре экрана находится поле с надписью "Имя пользователя".

**Шаг 1.5:** Кликните левой кнопкой мыши в поле "Имя пользователя".

**Шаг 1.6:** Введите свое имя пользователя с клавиатуры.

**Шаг 1.7:** Под полем "Имя пользователя" находится поле с надписью "Пароль".

**Шаг 1.8:** Кликните левой кнопкой мыши в поле "Пароль".

**Шаг 1.9:** Введите свой пароль с клавиатуры (символы скрыты точками).

**Шаг 1.10:** В нижней части формы входа находится кнопка "Войти" (синяя кнопка справа).

**Шаг 1.11:** Кликните левой кнопкой мыши на кнопку "Войти".

**Ожидаемый результат:** 
- Если имя пользователя и пароль верны, произойдет перенаправление на главную страницу системы
- Если имя пользователя или пароль неверны, появится сообщение об ошибке "Неверные учетные данные"

**Шаг 1.12:** Проверьте, что вы находитесь на главной странице системы. Это видно по наличию меню навигации и заголовка "Главная".

### Пост-условия
- Пользователь авторизован в системе
- Открыта главная страница системы
- Доступны все функции управления кормлением

## 2. Создание задачи кормления

### Цель
Создать новую задачу кормления для указанного бассейна с заданными параметрами.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница "Управление кормлением"
- В системе есть хотя бы один бассейн
- В системе есть хотя бы один тип корма
- В системе есть хотя бы один период кормления

### Шаги выполнения

**Шаг 2.1:** В меню навигации слева кликните на пункт "Кормление".

**Шаг 2.2:** Кликните левой кнопкой мыши на пункт "Кормление" в меню.

**Ожидаемый результат:** Откроется страница со списком задач кормления.

**Шаг 2.3:** В верхней части страницы "Кормление" находится кнопка "Создать задачу" (синяя кнопка).

**Шаг 2.4:** Кликните левой кнопкой мыши на кнопку "Создать задачу".

**Ожидаемый результат:** Откроется модальное окно "Создание задачи кормления" с полями для ввода.

**Шаг 2.5:** В модальном окне в верхней части формы находится поле выбора бассейна с надписью "Бассейн".

**Шаг 2.6:** Кликните левой кнопкой мыши на поле выбора бассейна.

**Ожидаемый результат:** Откроется список доступных бассейнов.

**Шаг 2.7:** Выберите нужный бассейн из списка, кликнув на него левой кнопкой мыши.

**Ожидаемый результат:** В поле будет отображаться выбранный бассейн.

**Шаг 2.8:** Под полем "Бассейн" находится поле выбора корма с надписью "Корм".

**Шаг 2.9:** Кликните левой кнопкой мыши на поле выбора корма.

**Ожидаемый результат:** Откроется список доступных типов корма.

**Шаг 2.10:** Выберите нужный тип корма из списка, кликнув на него левой кнопкой мыши.

**Ожидаемый результат:** В поле будет отображаться выбранный тип корма.

**Шаг 2.11:** Под полем "Корм" находится поле ввода массы корма с надписью "Масса (кг)".

**Шаг 2.12:** Кликните левой кнопкой мыши в поле ввода массы.

**Шаг 2.13:** Введите числовое значение массы корма в килограммах с клавиатуры (например, `0.1` для 100 грамм или `1.5` для 1.5 килограмм).

**Ожидаемый результат:** В поле отображается введенное значение.

**Шаг 2.14:** Под полем "Масса (кг)" находится поле выбора периода кормления с надписью "Период".

**Шаг 2.15:** Кликните левой кнопкой мыши на поле выбора периода.

**Ожидаемый результат:** Откроется список доступных периодов кормления с опцией "Другое".

**Шаг 2.16:** Выберите период из списка, кликнув на него левой кнопкой мыши. Если вы выбрали "Другое", появится дополнительное поле "Период (другое)" для ввода времени вручную.

**Ожидаемый результат:** В поле будет отображаться выбранный период. Если выбрано "Другое", отобразится поле для ввода времени.

**Шаг 2.17:** Если вы выбрали "Другое" в поле "Период", введите время кормления в поле "Период (другое)" в формате ЧЧ:ММ (например, `14:30`).

**Ожидаемый результат:** В поле отображается введенное время.

**Шаг 2.18:** В нижней части модального окна находятся две кнопки: "Отменить" (слева) и "Создать" (справа, синяя кнопка).

**Шаг 2.19:** Кликните левой кнопкой мыши на кнопку "Создать".

**Ожидаемый результат:** 
- Если все поля заполнены корректно, задача будет создана
- Модальное окно закроется
- Новая задача появится в списке задач кормления
- Если есть ошибки валидации, появятся сообщения об ошибках под соответствующими полями

**Шаг 2.20:** Проверьте, что новая задача отображается в списке задач кормления.

### Пост-условия
- Создана новая задача кормления
- Задача отображается в списке с статусом "Запланировано"
- Все введенные данные сохранены корректно

## 3. Просмотр списка задач кормления

### Цель
Просмотреть список всех созданных задач кормления.

### Предусловия
- Пользователь авторизован в системе
- В системе может быть хотя бы одна задача кормления (список может быть пустым)

### Шаги выполнения

**Шаг 3.1:** В меню навигации слева кликните на пункт "Кормление".

**Шаг 3.2:** Кликните левой кнопкой мыши на пункт "Кормление" в меню.

**Ожидаемый результат:** Откроется страница со списком задач кормления.

**Шаг 3.3:** Просмотрите таблицу со списком задач. Таблица должна содержать следующие колонки:
- Бассейн
- Корм
- Масса (кг)
- Период
- Кнопка удаления

**Шаг 3.4:** Если задач много и они не помещаются на одной странице, в нижней части таблицы находятся элементы пагинации (кнопки "Следующая", "Предыдущая", номера страниц).

**Шаг 3.5:** Для перехода на следующую страницу кликните на кнопку "Следующая" или номер следующей страницы в пагинации внизу таблицы.

**Ожидаемый результат:** Отобразятся задачи со следующей страницы.

**Шаг 3.6:** Для перехода на предыдущую страницу кликните на кнопку "Предыдущая" или номер предыдущей страницы в пагинации внизу таблицы.

**Ожидаемый результат:** Отобразятся задачи с предыдущей страницы.

**Шаг 3.7:** Для изменения количества задач на странице в пагинации внизу таблицы на странице "Кормление" кликните на выпадающий список с количеством элементов (например, "10", "20", "50").

**Шаг 3.8:** Кликните на выпадающий список в пагинации.

**Шаг 3.9:** Выберите нужное количество задач на странице.

**Ожидаемый результат:** Количество отображаемых задач изменится согласно выбранному значению.

### Пост-условия
- Отображен список всех задач кормления
- Пользователь может видеть все доступные задачи

## 4. Редактирование задачи кормления

### Цель
Изменить параметры существующей задачи кормления.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть хотя бы одна задача кормления

### Шаги выполнения

**Шаг 4.1:** Откройте раздел "Кормление" (в меню навигации слева кликните на пункт "Кормление") и в таблице найдите строку с задачей кормления, которую нужно отредактировать.

**Шаг 4.2:** В правой части строки задачи находится кнопка редактирования (иконка карандаша), или дважды кликните на ячейку, которую нужно изменить.

**Шаг 4.3:** Кликните левой кнопкой мыши на кнопку редактирования или дважды кликните на ячейку.

**Ожидаемый результат:** Ячейка станет редактируемой.

**Шаг 4.4:** Для редактирования ячейки:
- Кликните на ячейку, которую нужно изменить
- Введите новое значение или выберите из списка (для полей с выпадающим списком)
- Нажмите клавишу `Enter` или кликните вне ячейки для сохранения

**Шаг 4.5:** Изменения сохраняются автоматически после нажатия `Enter` или клика вне ячейки.

**Шаг 4.7:** Кликните левой кнопкой мыши на кнопку сохранения.

**Ожидаемый результат:** 
- Изменения сохранены
- Появится сообщение об успешном сохранении (если предусмотрено)
- Обновленные данные отображаются в таблице
- Если есть ошибки валидации, появятся сообщения об ошибках

**Шаг 4.8:** Проверьте, что изменения отображаются в списке задач.

### Пост-условия
- Задача кормления обновлена с новыми параметрами
- Изменения сохранены в системе
- Обновленные данные отображаются в списке

## 5. Удаление задачи кормления

### Цель
Удалить задачу кормления из системы.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть хотя бы одна задача кормления

### Шаги выполнения

**Шаг 5.1:** Откройте раздел "Кормление" (в меню навигации слева кликните на пункт "Кормление") и в таблице найдите строку с задачей кормления, которую нужно удалить.

**Шаг 5.2:** В правой части строки задачи находится кнопка удаления (иконка корзины).

**Шаг 5.3:** Кликните левой кнопкой мыши на кнопку удаления.

**Ожидаемый результат:** Задача будет удалена из системы.

**Шаг 5.4:** Проверьте, что задача исчезла из списка.

**Ожидаемый результат при подтверждении:** 
- Задача удалена из системы
- Появится сообщение об успешном удалении (если предусмотрено)
- Задача исчезнет из списка задач
- Если была ошибка, появится сообщение об ошибке

**Ожидаемый результат при отмене:** 
- Диалоговое окно закроется
- Задача останется в списке без изменений

**Шаг 5.6:** Проверьте, что удаленная задача больше не отображается в списке задач.

### Пост-условия
- Задача кормления удалена из системы
- Задача больше не отображается в списке
- Данные удалены из базы данных

## 6. Фильтрация задач кормления

### Цель
Отфильтровать список задач кормления по заданным критериям.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть задачи кормления

### Шаги выполнения

**Шаг 6.1:** На странице "Кормление" над таблицей задач находится панель фильтров с полями для фильтрации.

**Шаг 6.2:** Для фильтрации по бассейну:
- В панели фильтров над таблицей на странице "Кормление" кликните на поле фильтра "Бассейн" (первое поле слева)
- Кликните левой кнопкой мыши на это поле
- Выберите нужный бассейн из выпадающего списка

**Ожидаемый результат:** Список задач обновится и покажет только задачи для выбранного бассейна.

**Шаг 6.3:** Для фильтрации по типу корма:
- В панели фильтров над таблицей на странице "Кормление" кликните на поле фильтра "Корм" (второе поле)
- Кликните левой кнопкой мыши на это поле
- Выберите нужный тип корма из выпадающего списка

**Ожидаемый результат:** Список задач обновится и покажет только задачи с выбранным типом корма.

**Шаг 6.4:** Для фильтрации по массе корма:
- В панели фильтров над таблицей на странице "Кормление" кликните на слайдер фильтра "Масса" (третье поле)
- Переместите левый ползунок слайдера для установки минимального значения массы
- Переместите правый ползунок слайдера для установки максимального значения массы
- Или кликните на нужное значение на шкале слайдера

**Ожидаемый результат:** Список задач обновится и покажет только задачи с массой в указанном диапазоне.

**Шаг 6.5:** Для сброса фильтров:
- Очистите выбранные значения в полях фильтров, кликнув на крестик в каждом поле
- Или обновите страницу

**Ожидаемый результат:** Все фильтры сброшены, отображаются все задачи без фильтрации.

**Шаг 6.6:** Проверьте, что список задач соответствует примененным фильтрам.

### Пост-условия
- Применены выбранные фильтры
- Список задач отображает только задачи, соответствующие критериям фильтрации

## 7. Экспорт данных о кормлении

### Цель
Выгрузить данные о задачах кормления в файл для дальнейшего анализа.

### Предусловия
- Пользователь авторизован в системе
- Открыта страница со списком задач кормления
- В системе есть задачи кормления

### Шаги выполнения

**Шаг 7.1:** На странице "Кормление" в панели инструментов над таблицей находится кнопка экспорта.

**Шаг 7.2:** Кликните левой кнопкой мыши на кнопку экспорта.

**Ожидаемый результат:** 
- Если применены фильтры, экспортируются только отфильтрованные данные
- Если фильтры не применены, экспортируются все данные
- Начнется загрузка файла

**Шаг 7.3:** Дождитесь завершения загрузки файла. Файл автоматически сохраняется в папку "Загрузки" (Downloads).

**Шаг 7.4:** Откройте папку "Загрузки" (Downloads) на вашем компьютере и найдите загруженный файл `feeding_data.csv`.

**Шаг 7.5:** Откройте файл в программе для работы с таблицами (Microsoft Excel, LibreOffice Calc или текстовом редакторе).

**Ожидаемый результат:** Файл содержит данные о задачах кормления в формате CSV с колонками:
- Бассейн
- Популяция
- Время
- Масса
- Тип корма

**Шаг 7.6:** Проверьте, что все данные экспортированы корректно и соответствуют данным в системе.

### Пост-условия
- Файл с данными о кормлении сохранен на компьютере пользователя
- Данные в файле соответствуют данным в системе
- Файл можно открыть и проанализировать в сторонних программах

## 8. Просмотр логов системы

### Цель
Просмотреть журнал событий системы для мониторинга работы оборудования и выявления проблем.

### Предусловия
- Пользователь авторизован в системе
- В системе есть записи в журнале логов

### Шаги выполнения

**Шаг 8.1:** В меню навигации слева кликните на пункт "Логи".

**Шаг 8.2:** Кликните левой кнопкой мыши на пункт "Логи" в меню.

**Ожидаемый результат:** Откроется страница со списком логов системы.

**Шаг 8.3:** Просмотрите таблицу со списком логов. Таблица содержит следующие колонки:
- Тип - может быть "Тележка", "Бункер", "Система"
- Действие - описание выполненного действия
- Время - дата и время события
- Описание - дополнительная информация о событии

**Шаг 8.4:** Логи отсортированы по времени, самые свежие сверху.

## 13. Проверка движения тележки

### Цель
Проверить работоспособность системы перемещения тележки и точность ее позиционирования.

### Предусловия
- Система включена и готова к работе
- Тележка находится в исходном положении (домашняя позиция)
- Рельсы свободны от препятствий
- Оператор имеет доступ к физическому оборудованию
- Пользователь авторизован в веб-интерфейсе

### Шаги выполнения

**Шаг 13.1:** Визуально осмотрите тележку и рельсы на наличие видимых повреждений, препятствий или посторонних предметов.

**Ожидаемый результат:** Тележка и рельсы находятся в исправном состоянии, препятствий нет.

**Шаг 13.2:** Проверьте подключение питания тележки. Убедитесь, что кабели подключены надежно, нет повреждений изоляции.

**Ожидаемый результат:** Питание подключено корректно, индикаторы питания (если есть) горят.

**Шаг 13.3:** В веб-интерфейсе откройте раздел "Тестирование" (в меню навигации слева кликните на пункт "Тестирование"), затем перейдите на вкладку "Движение тележки".

**Шаг 13.4:** На вкладке "Движение тележки" в разделе "Тестирование" кликните на кнопку "Движение вперед" для запуска перемещения тележки.

**Ожидаемый результат:** Тележка начинает движение по рельсам в направлении целевого бассейна.

**Шаг 13.5:** Наблюдайте за движением тележки. Проверьте:
- Плавность движения (нет рывков, заеданий)
- Равномерность скорости
- Отсутствие посторонних звуков (скрежет, стук)
- Отсутствие вибраций

**Ожидаемый результат:** Тележка движется плавно, равномерно, без посторонних звуков и вибраций.

**Шаг 13.6:** Засеките время движения тележки от исходной позиции до целевого бассейна с помощью секундомера или часов.

**Ожидаемый результат:** Время перемещения не превышает 30 секунд (согласно требованиям).

**Шаг 13.7:** Когда тележка достигнет целевого бассейна, визуально оцените точность позиционирования. Измерьте расстояние от тележки до центра бассейна с помощью рулетки или линейки.

**Ожидаемый результат:** Тележка остановилась с точностью не более 5 см от целевого бассейна.

**Шаг 13.8:** Проверьте в веб-интерфейсе, что система зафиксировала достижение тележкой целевой позиции. Откройте раздел "Логи" (в меню навигации слева кликните на пункт "Логи") и проверьте наличие записи о движении тележки.

**Ожидаемый результат:** В логах зафиксировано событие достижения тележкой целевой позиции.

**Шаг 13.9:** Запустите возврат тележки в исходное положение (домашнюю позицию).

**Ожидаемый результат:** Тележка начинает движение в обратном направлении.

**Шаг 13.10:** Наблюдайте за возвратом тележки. Проверьте те же параметры, что и при движении вперед.

**Ожидаемый результат:** Тележка плавно возвращается в исходное положение.

**Шаг 13.11:** Засеките время возврата тележки в исходное положение.

**Ожидаемый результат:** Время возврата не превышает 30 секунд.

**Шаг 13.12:** Проверьте, что тележка точно остановилась в исходном положении. Визуально оцените позицию относительно маркеров или концевых выключателей.

**Ожидаемый результат:** Тележка остановилась точно в исходном положении.

**Шаг 13.13:** Повторите проверку перемещения к другому бассейну (на другом конце рельсов) для проверки работы на всей длине маршрута.

**Ожидаемый результат:** Тележка успешно перемещается к бассейну на другом конце рельсов с той же точностью.

### Пост-условия
- Тележка проверена на работоспособность
- Подтверждена точность позиционирования (не более 5 см)
- Подтверждено время перемещения (не более 30 секунд)
- Все движения зафиксированы в логах системы
- Тележка возвращена в исходное положение

**Шаг 15.6:** Положите на платформу эталонный груз известной массы (например, 100 грамм). Подождите 2-3 секунды для стабилизации показаний.

**Ожидаемый результат:** Весы показывают массу, близкую к массе эталонного груза. Погрешность измерения не превышает 5 г (согласно требованиям).

**Шаг 15.7:** Снимите груз с платформы. Проверьте, что показания вернулись к нулю (или близко к нулю).

**Ожидаемый результат:** Показания весов вернулись к нулю.

**Шаг 15.8:** Проверьте работу весов при загрузке корма через шнековый механизм (см. раздел 14). Следите за показаниями весов в реальном времени на вкладке "Весы" в разделе "Тестирование".

**Ожидаемый результат:** 
- Показания весов обновляются в реальном времени (данные приходят с Arduino)
- Масса увеличивается плавно по мере подачи корма
- При достижении заданной массы система корректно останавливает подачу
- Фактическая масса соответствует заданной с погрешностью не более 3%

**Шаг 15.9:** Откройте раздел "Логи" (в меню навигации слева кликните на пункт "Логи") и проверьте, что все измерения массы зафиксированы с отметками времени.

**Ожидаемый результат:** В логах зафиксированы все измерения массы с указанием времени.

## 16. Проверка RFID

### Цель
Проверить работоспособность RFID-системы для идентификации бассейнов и бункеров.

### Предусловия
- Система включена и готова к работе
- Тележка находится в рабочем положении
- На бассейнах и бункерах установлены RFID-метки
- Пользователь авторизован в веб-интерфейсе
- Оператор имеет доступ к физическому оборудованию

### Шаги выполнения

**Шаг 16.1:** Визуально осмотрите RFID-ридер на тележке на наличие видимых повреждений или посторонних предметов.

**Ожидаемый результат:** RFID-ридер находится в исправном состоянии.

**Шаг 16.2:** Проверьте подключение RFID-ридера к системе. Убедитесь, что кабели подключены надежно, нет повреждений изоляции.

**Ожидаемый результат:** RFID-ридер подключен корректно.

**Шаг 16.3:** Проверьте состояние RFID-меток на бассейнах и бункерах. Убедитесь, что метки:
- Не повреждены
- Правильно установлены
- Находятся в зоне действия ридера

**Ожидаемый результат:** Все RFID-метки находятся в исправном состоянии.

**Шаг 16.4:** В веб-интерфейсе откройте раздел "Тестирование" (в меню навигации слева кликните на пункт "Тестирование"), затем перейдите на вкладку "RFID".

**Шаг 16.5:** На вкладке "RFID" в карточке "Тест RFID" введите идентификатор бассейна в поле "ID (опционально)" и кликните на кнопку "Тест считывания" для проверки работы RFID-ридера.

**Ожидаемый результат:** Тележка начинает движение к целевому бассейну.

**Шаг 16.6:** Когда тележка достигнет бассейна, наблюдайте за процессом считывания RFID-метки. Проверьте, что ридер находится в зоне действия метки и пытается ее считать.

**Ожидаемый результат:** RFID-ридер активируется и пытается считать метку.

**Шаг 16.7:** Откройте раздел "Логи" (в меню навигации слева кликните на пункт "Логи") и проверьте, что RFID-метка была успешно считана и идентифицирована (должна быть запись о тестировании RFID).

**Ожидаемый результат:** 
- В логах зафиксировано событие успешного считывания RFID-метки
- Система корректно идентифицировала бассейн
- Параметры кормления установлены согласно данным бассейна

**Шаг 16.8:** Повторите проверку для нескольких разных бассейнов с разными RFID-метками.

**Ожидаемый результат:** Все RFID-метки успешно считываются и идентифицируются.

**Шаг 16.9:** Проверьте работу RFID-ридера при считывании меток на бункерах (при загрузке корма).

**Ожидаемый результат:** RFID-ридер успешно считывает метки на бункерах и идентифицирует тип корма.

**Шаг 16.10:** Проверьте реакцию системы при нечитаемой RFID-метке (симуляция):
- Уберите RFID-метку (если это возможно)
- Или используйте поврежденную метку (если есть)
- Попробуйте переместить тележку к этой позиции

**Ожидаемый результат:** 
- Система обнаруживает проблему считывания
- Система оповещает оператора через веб-интерфейс
- Система переходит в безопасный режим, предотвращающий дальнейшее перемещение

**Шаг 16.11:** Восстановите RFID-метку и проверьте, что система вернулась в нормальный режим работы.

**Ожидаемый результат:** Система корректно работает после восстановления RFID-метки.

**Шаг 16.12:** Проверьте точность идентификации. Убедитесь, что система не путает разные бассейны или бункера.

**Ожидаемый результат:** Система всегда корректно идентифицирует бассейны и бункера, не путая их.

### Пост-условия
- RFID-система проверена на работоспособность
- Подтверждена корректная идентификация бассейнов и бункеров
- Подтверждена корректная реакция системы на проблемы считывания
- Все события зафиксированы в логах системы

- Система включена и готова к работе
- Тележка находится в исходном положении
- В бункерах есть корм для тестирования
- Пользователь авторизован в веб-интерфейсе
- Оператор имеет доступ к физическому оборудованию

**Шаг 17.1:** Выполните визуальный осмотр всей системы:
- Проверьте состояние тележки

- Кликните на кнопку "Создать" в нижней части модального окна

- Механизм останавливается при достижении заданной массы с погрешностью не более 3%
